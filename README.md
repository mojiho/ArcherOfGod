Project: Divine Archer (Archer of God 모작)

📖 프로젝트 소개
모바일 대전 액션 게임 **'아처 오브 갓(Archer of God)'**의 핵심 코어 루프를 분석하고 재구현한 개인 프로젝트입니다.

원작 특유의 긴장감 넘치는 활쏘기 메커니즘과 스타일리시한 비주얼을 구현하는 데 중점을 두었으며, 특히 실시간 PvP 환경을 고려한 설계와 모바일 환경에서의 **성능 최적화(Zero-GC 지향)**에 기술적 목표를 두었습니다.

핵심 기술 구현 
1. 물리 기반 투사체 시스템과 궤적 예측 (Deterministic Physics)
'아처 오브 갓'의 핵심 재미는 바람과 중력을 계산하여 상대를 맞추는 것입니다. 이를 위해 유니티 내장 물리 엔진(PhysX)에 전적으로 의존하지 않고, 네트워크 동기화를 염두에 둔 결정론적(Deterministic) 물리 계산 로직을 자체 구현했습니다.

Custom Trajectory Simulation: 중력 가속도와 풍향 벡터를 적용하여 매 프레임 화살의 위치를 직접 계산합니다.

Trajectory Prediction Line: 플레이어가 조준 시, 실제 발사될 화살의 예상 경로를 LineRenderer를 통해 시각적으로 오차 없이 표현합니다. 시뮬레이션 로직과 실제 발사 로직이 동일한 수식을 공유하여 정확도를 보장합니다.

2. URP 기반 스타일리시 카툰 렌더링
원작의 어둡고 스타일리시한 분위기를 내기 위해 URP 환경에서 커스텀 셰이딩을 적용했습니다.

Custom Toon Shader: Shader Graph를 활용하여 빛의 방향에 따른 셀 셰이딩(Cel-shading) 효과와 아웃라인(Outline)을 구현하여 캐릭터의 입체감을 강조했습니다.

Post-Processing 최적화: 모바일 성능에 큰 영향을 주는 블룸(Bloom) 등의 후처리 효과를 최소한의 비용으로 연출하기 위해, 특정 레이어에만 선별적으로 효과를 적용하거나 셰이더 단계에서 이미시브(Emissive) 효과를 구현했습니다.

3. 극한의 성능 최적화 (Performance-First)
실시간 대전 게임에서 프레임 드랍은 치명적입니다. 가비지 컬렉션(GC) 스파이크를 원천 차단하기 위한 설계를 적용했습니다.

Advanced Object Pooling: 화살, 히트 이펙트, UI 데미지 텍스트 등 빈번하게 생성/파괴되는 모든 오브젝트를 자체 구현한 풀링 시스템으로 관리합니다.

Struct & Span 활용: 빈번한 데이터 교환 시 class 대신 struct를 적극 활용하고, 배열 처리 시 Span<T>이나 ArraySegment<T>를 사용하여 불필요한 힙 할당을 방지했습니다.

Update 최소화: 매 프레임 호출되는 Update 문 사용을 지양하고, UniTask를 활용한 비동기 로직이나 이벤트 기반 처리를 통해 CPU 부하를 줄였습니다.
